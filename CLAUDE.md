## Layout

- `BroadcastBrain.xcodeproj/` — Xcode project (generated by `xcodegen` from `project.yml`)
- `BroadcastBrain/` — SwiftUI app source
- `BroadcastBrainTests/` — XCTest suite
- `BroadcastBrain/Frameworks/cactus-macos.xcframework` — prebuilt Cactus SDK (includes `Modules/module.modulemap`)
- `reference/` — main-branch design handoff (read-only, excluded from app bundle)

## Build + run

```bash
xcodegen generate                                   # regenerate after editing project.yml
xcodebuild build -project BroadcastBrain.xcodeproj -scheme BroadcastBrain -destination 'platform=macOS'
xcodebuild test  -project BroadcastBrain.xcodeproj -scheme BroadcastBrain -destination 'platform=macOS'
open -a Xcode BroadcastBrain.xcodeproj              # for ⌘R
```

## Demo modes

- `MOCK_MODE=1` env var (set in the scheme by default) → `MockResponder` surfaces 5 scripted stat cards matched against transcript substrings. Use this for the demo.
- Unset → the app tries to load Gemma weights from `~/Library/Application Support/BroadcastBrain/models/gemma.gguf`. Falls back to `MockResponder` if missing.

## Non-negotiables

- macOS 14 target. iPad port is a follow-up.
- On-device only. No network during demo.
- Every stat card shows `Sportradar ✓`. Null Gemma response → "No verified data" card.
- Port JSX visuals 1:1 to SwiftUI. Do not redesign.

## Cactus XCFramework provenance

The xcframework was built locally from `github.com/cactus-compute/cactus` @ `v1.14` via `apple/build.sh` (~97 seconds on M-series). `Cactus.swift` was copied from the same repo's `apple/Cactus.swift`. A `Modules/module.modulemap` was manually installed into `cactus.framework/` from the repo's `apple/module.modulemap` — the build script does not copy it automatically.

# Gemma 4 Install Flow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Gemma 4 E2B (via Cactus) reliably usable by BroadcastBrain — no silent failures, no zombie UI state, tolerable first-run install time, and a bulletproof path for the YC Gemma/Cactus hackathon demo.

**Architecture:** Three-phase fix. Phase 1 fixes the lying UI by gating the agent on model readiness and showing a real install screen. Phase 2 mirrors the 4.68 GB int4 zip to our own CDN and patches cactus-react-native's registry to point at it, for faster + more reliable downloads on unknown networks. Phase 3 is a runbook — pre-warm the demo device + a one-page checklist. Phases are independent and each one ships value on its own.

**Tech Stack:** React Native 0.74+, Expo (ejected), zustand (existing), cactus-react-native `^1.13.x`, patch-package (new), Cloudflare R2 or CloudFront (new, out-of-repo).

## Context — what we know from the code

- `app/src/cactus/client.ts:11` — `DEFAULT_MODEL = 'gemma-4-e2b-it'`
- `app/src/cactus/state/modelLoader.ts` — zustand store, states: `'unloaded' | 'downloading' | 'loading' | 'ready' | 'error'`. Retry with exponential backoff already in place (5 attempts, 2s/4s/8s/16s).
- `app/App.tsx:69` — `ensureModel()` fires in `useEffect` on mount. No gate on screens.
- `app/src/cactus/devtools/ModelPill.tsx` — tiny top-center pill showing status. Error state truncates to `GEMMA · ERR`.
- `app/src/agent/AgentContext.tsx:68` — `start()` has no model-readiness guard. Calling `start()` with `status !== 'ready'` flips `active=true` and the UI renders "Agent is listening" while Gemma is actually broken.
- Cactus SDK behavior (`node_modules/cactus-react-native/src/modelRegistry.ts`):
  - Registry URL is hardcoded: `https://huggingface.co/api/models?author=Cactus-Compute`
  - Resolves `gemma-4-e2b-it` → `https://huggingface.co/Cactus-Compute/gemma-4-E2B-it/resolve/<version>/weights/gemma-4-e2b-it-int4.zip` (4.68 GB). With `pro: true` it would use `-int4-apple.zip`.
  - `CactusLM.isModelPath(s)` returns true for strings starting with `file://` or `/`. Passing such a path skips the download + registry entirely and calls `init(modelPath)` directly. We don't use that today but it's a viable escape hatch.
  - `downloadModel` is implemented in the iOS native module — no resume, no range requests. A retry restarts at byte 0.

## Non-negotiable constraints

- **Must use Gemma via Cactus.** (YC hackathon theme requirement; user-confirmed.)
- **Cannot bundle the model in the IPA.** The int4 zip alone is 4.68 GB; App Store caps IPAs at 4 GB.
- **Must run on iOS first.** TestFlight is the distribution channel.

import React from 'react';
import { Pressable, Text, View } from 'react-native';
import { FONT_MONO, tokens } from '../theme/tokens';
import { useModelLoader } from '../cactus/state/modelLoader';

const MODEL_SIZE_GB = 4.68;

export function ModelInstallScreen() {
  const status   = useModelLoader((s) => s.status);
  const progress = useModelLoader((s) => s.progress);
  const error    = useModelLoader((s) => s.error);
  const reset    = useModelLoader((s) => s.reset);
  const retry    = useModelLoader((s) => s.ensureLoaded);

  const pct = Math.max(0, Math.min(100, Math.round(progress * 100)));

  return (
    <View style={{ flex: 1, backgroundColor: tokens.bgBase, alignItems: 'center', justifyContent: 'center', padding: 32 }}>
      <Text style={{ fontFamily: FONT_MONO, fontSize: 10, letterSpacing: 2.4, color: tokens.textMuted, fontWeight: '700', marginBottom: 10 }}>

      <Text style={{ fontFamily: FONT_MONO, fontSize: 22, fontWeight: '700', color: tokens.text, marginBottom: 6 }}>
        {status === 'error' ? 'Install failed' : 'Installing the on-device model'}
      </Text>
      <Text style={{ fontFamily: FONT_MONO, fontSize: 12, color: tokens.textMuted, marginBottom: 24, textAlign: 'center', maxWidth: 420 }}>
        {status === 'error'
          ? 'The download couldn\u2019t finish. Check your connection and try again. The model is ~4.68 GB and only needs to install once.'
          : `BroadcastBrain runs Gemma 4 locally for zero-latency voice. One-time ~${MODEL_SIZE_GB} GB download — leave the app open on Wi\u2011Fi.`}

          <View style={{ width: 420, height: 10, borderWidth: 1, borderColor: tokens.border, borderRadius: 4, overflow: 'hidden', marginBottom: 10 }}>
            <View style={{ width: `${pct}%`, height: '100%', backgroundColor: tokens.esoteric }} />
          </View>
          <Text style={{ fontFamily: FONT_MONO, fontSize: 11, color: tokens.textMuted }}>
            {pct}% &nbsp;·&nbsp; {downloadedGb} / {MODEL_SIZE_GB} GB
          </Text>
          {!!error && (
            <Text style={{ fontFamily: FONT_MONO, fontSize: 10, color: tokens.textMuted, marginTop: 12, maxWidth: 420, textAlign: 'center' }} numberOfLines={2}>
              {error}
            </Text>
          )}
        </>

          <Text style={{ fontFamily: FONT_MONO, fontSize: 10, color: tokens.live, marginBottom: 18, maxWidth: 420, textAlign: 'center' }} numberOfLines={3}>

## Phase 2 — Mirror the model on our own CDN

### Context

Cactus's `modelRegistry.ts` builds HF URLs at runtime. To change the origin we either fork the package or use `patch-package`. The latter is the right tool for a hackathon: a tiny diff, checked in, re-applied on every `npm install`.

The plan: upload `gemma-4-e2b-it-int4.zip` to our own bucket (Cloudflare R2 is free for the bandwidth we need), then patch `modelRegistry.ts` to rewrite the HF URL for that specific model to our mirror URL.

Create `scripts/mirror-gemma-weights.sh`:

```sh
#!/usr/bin/env bash
# mirror-gemma-weights.sh — one-shot mirror of Cactus-Compute Gemma 4 E2B weights to R2.
# Requires: curl, aws CLI (with S3 plugin), env vars above.
set -euo pipefail

MODEL_ID="Cactus-Compute/gemma-4-E2B-it"
FILES=(
  "gemma-4-e2b-it-int4.zip"
  "gemma-4-e2b-it-int4-apple.zip"
)
VERSION="${MODEL_VERSION:-main}"
BUCKET="broadcastbrain-models"

TMP="$(mktemp -d)"
trap 'rm -rf "$TMP"' EXIT

for f in "${FILES[@]}"; do
  echo "→ downloading $f"
  curl -fL "https://huggingface.co/${MODEL_ID}/resolve/${VERSION}/weights/${f}" -o "${TMP}/${f}"
  echo "→ uploading to R2: ${f}"
  aws s3 cp "${TMP}/${f}" "s3://${BUCKET}/weights/${f}" \
    --endpoint-url "https://${R2_ACCOUNT_ID}.r2.cloudflarestorage.com" \
    --content-type application/zip
done

echo "done. verify at: ${R2_PUBLIC_BASE}/weights/${FILES[0]}"
```

Make it executable:
